# Клиенты

[← README](../README.md)


Добавить клиента:

```bash
xray-client add ИМЯ
```

После команды можно ввести количество календарных дней доступа.
Пустой ввод или `0` означает бессрочный доступ.
Клиент со сроком будет автоматически отключён в `00:00` по часовому поясу менеджера после окончания срока.

Добавить клиента сразу на конкретный срок:

```bash
xray-client add ИМЯ ДНЕЙ
```

Добавить клиента в конкретное подключение:

```bash
xray-client add ИМЯ ДНЕЙ --connection TAG
```

Тип создаваемого credential зависит от выбранного подключения. Для VLESS-подключения создаётся UUID и `vless://` ссылка. Для Trojan TLS/WebSocket-подключения через Caddy создаётся внутренний UUID менеджера, отдельный Trojan password и `trojan://` ссылка; в Xray config активный пользователь хранится в `settings.clients`.
Внутренний UUID клиента хранится отдельно от protocol credential: один клиент может иметь несколько credentials, например VLESS Reality, VLESS TLS/XHTTP и Trojan TLS/WebSocket. Для VLESS credential используется свой UUID в `vless://` ссылке, для Trojan credential - свой password в `trojan://` ссылке. Внутренний UUID клиента нужен менеджеру для SQLite, маршрутизации, Telegram-подписок и клиентского ключа `vpn-key:UUID`; он не является Trojan-паролем.
В меню `Клиенты -> Добавить клиента` выбор подключения показывает и VLESS, и Trojan connections, поэтому клиента можно сразу добавить в нужный протокол.

Для существующего клиента используй отдельный пункт меню:

```text
Клиенты -> Добавить подключение к клиенту
```

Меню сначала показывает таблицу клиентов, затем только те VLESS/Trojan connections, которых у выбранного клиента ещё нет. Если все подключения уже добавлены, действие отменяется без изменения клиента.

Через CLI совместимый способ остаётся прежним: повторный вызов `xray-client add ИМЯ ... --connection TAG` добавляет существующему клиенту ещё один credential в выбранном подключении. Дублировать один и тот же `connection` для одного клиента нельзя.

Новый клиент по умолчанию бесплатный и не участвует в расчёте оплаты. Чтобы добавить платного клиента через команду:

```bash
xray-client add ИМЯ ДНЕЙ --payment paid
```

Изменить статус оплаты существующего клиента:

```bash
xray-client set-payment ИМЯ paid
xray-client set-payment ИМЯ free
```

После добавления клиента или смены статуса оплаты команда выводит текущий расчёт суммы на одного платного клиента.

Продлить срок доступа существующего клиента на N дней:

```bash
xray-client extend-days ИМЯ ДНЕЙ
```

При продлении дни прибавляются к текущей дате окончания доступа. Если срок уже истёк или не был установлен, продление считается от сегодняшней даты.

Установить срок доступа заново от сегодняшней даты:

```bash
xray-client set-days ИМЯ ДНЕЙ
```

Сделать доступ бессрочным:

```bash
xray-client set-days ИМЯ 0
```

После изменения срока менеджер сразу пересчитывает статус клиента. Если срок истёк, клиент отключается. Если срок продлён или сделан бессрочным, клиент включается обратно, но только если не исчерпан дневной или месячный лимит трафика. Лимит трафика имеет приоритет над сроком доступа.

Временно отключить клиента без удаления:

```bash
xray-client disable ИМЯ
```

Отключить только один credential клиента:

```bash
xray-client disable ИМЯ --connection TAG
```

Включить клиента обратно с той же ссылкой:

```bash
xray-client enable ИМЯ
```

Включить только один credential:

```bash
xray-client enable ИМЯ --connection TAG
```

Перенести клиента в другое подключение:

```text
Клиенты -> Перенести клиента в другое подключение
```

Меню сначала попросит выбрать клиента из таблицы, затем целевое VLESS-подключение.
Для включенного клиента менеджер перемещает UUID между inbounds в `config.json`, проверяет Xray config, перезапускает Xray и печатает новую VLESS-ссылку.
Для отключенного клиента меняется сохранённая привязка в SQLite без перезапуска Xray.
Перенос между VLESS и Trojan подключениями не нужен для multi-credential сценария: вместо переноса добавь клиенту credential в другом подключении. Старый `move-connection` остаётся для переноса одиночного VLESS credential между VLESS connections.

То же через командную строку:

```bash
xray-client move-connection ИМЯ CONNECTION_NAME_OR_TAG
```

После переноса нужно выдать клиенту новую ссылку, потому что порт, SNI, transport или `security` могут измениться.

Полностью удалить клиента:

```bash
xray-client remove ИМЯ
```

Повторно вывести ссылку клиента:

```bash
xray-client link ИМЯ
```

Если у клиента несколько credentials, команда выводит список ссылок по подключениям. Чтобы получить одну конкретную ссылку:

```bash
xray-client link ИМЯ --connection TAG
```

Вывести клиентский ключ для Telegram-подписки:

```bash
xray-client key ИМЯ
```

Ключ имеет вид `vpn-key:UUID` и привязан к клиенту целиком, а не к одному credential.

Синхронизировать per-client cascade routing rules и balancers в `config.json`:

```bash
xray-client sync-routes
```

После начальной синхронизации переключить выбранную страну/каскад клиента без изменения VLESS-ссылки:

```bash
xray-client route ИМЯ
xray-client route ИМЯ Германия
xray-client route ИМЯ cascade-us
```

`route` сохраняет выбранный `cascade-*` в SQLite и через Xray API пытается переключить runtime balancer без перезапуска Xray. Если выбран тот же cascade, команда сообщает, что страна уже выбрана.

Список клиентов:

```bash
xray-client list
```

В списке показываются две framed-таблицы. `Clients` агрегирует статус, оплату, страну каскада, online/offline, общий трафик, лимит, срок доступа и количество credentials клиента. `Credentials` показывает конкретные protocol credentials с protocol/security/transport/connection, отдельным online state, traffic и временем обновления traffic. Для Caddy-managed подключений `SECURITY` отображается как `tls/caddy`: публичное клиентское подключение идёт по TLS через Caddy, а локальный inbound Xray остаётся без TLS.

```text
Clients
+-------+---------+---------+---------+--------+----------------------+----------+-------------+----------------------+------------------+-------+
| NAME  | STATUS  | PAYMENT | COUNTRY | ONLINE | LAST ONLINE          | TOTAL    | LIMIT       | TRAFFIC UPDATED      | ACCESS UNTIL     | CREDS |
+-------+---------+---------+---------+--------+----------------------+----------+-------------+----------------------+------------------+-------+
| alice | enabled | paid    | Германия | online | 2026-06-23 12:31 UTC | 12.00GB  | без лимита  | 2026-06-23 12:31 UTC | 2026-07-23 00:00 | 2/2   |
+-------+---------+---------+---------+--------+----------------------+----------+-------------+----------------------+------------------+-------+

Credentials
+--------+----------+-----------+-----------+-------------+---------+--------+---------+----------------------+----------------------+
| CLIENT | PROTOCOL | SECURITY  | TRANSPORT | CONNECTION  | STATUS  | ONLINE | TRAFFIC | TRAFFIC UPDATED      | LAST ONLINE          |
+--------+----------+-----------+-----------+-------------+---------+--------+---------+----------------------+----------------------+
| alice  | vless    | reality   | tcp       | vless-main  | enabled | online | 8.00GB  | 2026-06-23 12:31 UTC | 2026-06-23 12:31 UTC |
| alice  | trojan   | tls/caddy | ws        | trojan-main | enabled | online | 4.00GB  | 2026-06-23 12:28 UTC | 2026-06-23 12:28 UTC |
+--------+----------+-----------+-----------+-------------+---------+--------+---------+----------------------+----------------------+
```

`IN` - трафик от клиента к серверу, `OUT` - трафик от сервера к клиенту.
`PAYMENT` показывает `free` или `paid`. Только `paid` участвует в расчёте оплаты и получает напоминания об оплате.
`COUNTRY` показывает текущую выбранную страну каскада для клиента.
Значения показываются в удобных единицах: `KB`, `MB` или `GB`.
`ONLINE` показывает `online`, если за последние 5 минут было зафиксировано принятое подключение или прирост трафика.
`TRAFFIC UPDATED` - последнее время, когда у клиента изменился `IN` или `OUT`.
`LAST ONLINE` - последнее время, когда Xray зафиксировал подключение клиента к серверу.
`ACCESS UNTIL` показывает дату и время отключения в часовом поясе менеджера или зелёную надпись `бессрочно`, если срок не установлен.
`LIMIT` показывает `без лимита`, дневной лимит `/day` или месячный лимит `/month`.

Накопительная статистика сохраняется в `/usr/local/etc/xray/manager.db` и переживает перезапуск Xray или сервера.
`xray-traffic-sync.timer` сохраняет счётчики раз в минуту, а `xray.service` пытается сохранить их перед штатной остановкой.
История по часам и дням хранится 6 месяцев, после этого старые дневные bucket удаляются автоматически.
Общий суммарный трафик клиента (`IN`, `OUT`, `TOTAL` в `xray-client list`) хранится постоянно.
Отчёты используют часовой пояс менеджера.
После смены часового пояса новые почасовые и дневные bucket статистики будут записываться уже по новой зоне.
Статистика начинает считаться после включения Xray stats и не восстанавливает трафик, который прошёл раньше.
