# Telegram бот

[← README](../README.md)


Telegram-бот отправляет админские GeoIP-уведомления в сохранённый приватный `chat_id` владельца. Клиенты могут отдельно подписаться на личные напоминания об окончании срока доступа, отправив боту свою VLESS-ссылку Reality или TLS/XHTTP, а затем включить или отключить для себя личные уведомления активности через кнопку `Уведомления активности`. Этот экран объясняет, что уведомления нужны для GeoIP/split-tunneling проверок и показывают только метаданные подключения: регион правила, адрес или домен, порт и время. Клиентские уведомления активности показывают только события этого клиента и не раскрывают внутреннее имя клиента; если владелец бота также подписан как клиент, клиентская копия в owner-чат не дублируется. Если предупреждений немного, клиентское сообщение показывает кнопку `Добавить в исключения`: пользователь выбирает адрес из списка, и бот больше не присылает ему личные предупреждения по этой цели. В меню `Уведомления активности` есть кнопка `Исключения`, где пользователь может посмотреть и удалить случайно добавленные личные исключения. Если целей или событий много, кнопка исключения не показывается; сообщение просит настроить split tunneling в VPN-клиенте, например отправить нужный GeoIP-регион, домены или IP в Direct/Bypass.

Первичная настройка:

```bash
xray-telegram setup
```

Во время настройки нужно вставить token от BotFather, отправить `/start` своему боту и выбрать найденный приватный чат. После настройки бот отправит тестовое сообщение.

Если при выборе `cascade` SSH-сессия оборвалась на перезапуске Xray, подключитесь к серверу заново и завершите привязку владельца:

```bash
xray-telegram owner
```

Эта команда использует уже сохранённый token и не перезапускает Xray. Если cascade-маршрут уже успел записаться в Xray config, команда предложит подхватить его для Telegram-бота без повторной перенастройки. В меню это пункт `Telegram бот` -> `Донастроить владельца/чат`.

Показать статус:

```bash
xray-telegram status
```

Выбрать маршрут Telegram Bot API:

```bash
xray-telegram mode direct
xray-telegram mode cascade
```

`direct` отправляет запросы Telegram напрямую с сервера. `cascade` создаёт локальный SOCKS inbound `127.0.0.1:10810` и направляет только Telegram Bot API через активный именованный `cascade-*`; если активный каскадный маршрут сейчас отключён, используется первый доступный cascade outbound.

Отправить тестовое сообщение:

```bash
xray-telegram test
```

Проверить новые GeoIP-события и отправить уведомление вручную:

```bash
xray-telegram notify-geoip
```

Обработать сообщения пользователей и привязать клиентские подписки:

```bash
xray-telegram poll-users
```

Обычно вручную запускать не нужно: быстрые ответы пользователям выполняет постоянный `xray-telegram-poller.service` через Telegram long polling.

Изменить имя бота в сообщениях пользователям:

```bash
xray-telegram bot-name "ИМЯ_БОТА"
```

Показать ежедневную сводку сервера вручную:

```bash
xray-telegram daily-summary --dry-run
xray-telegram notify-daily-summary --dry-run
```

Автоматически сводка отправляется владельцу бота один раз в день после 08:00 по часовому поясу менеджера. Трафик в сводке считается за предыдущий календарный день. В сводке также показываются месячная аренда сервера, годовая аренда домена, месячная доля домена и итоговая месячная аренда из настроек оплаты, размер SQLite-базы `manager.db` вместе с временными файлами `manager.db-wal`/`manager.db-shm`, если они есть, и статус необходимости перезагрузки сервера по `/var/run/reboot-required`. Если перезагрузка нужна из-за пакетов ядра, сводка показывает это отдельной формулировкой и перечисляет пакеты из `/var/run/reboot-required.pkgs`.


Проверить напоминания об оплате вручную:

```bash
xray-telegram notify-expiry
```

Настроить месячную аренду сервера, годовую аренду домена и округление:

```text
Telegram бот -> Настроить оплату и округление
```

Меню попросит ввести месячную аренду сервера, выбрать валюту: рубли `₽`, доллары `$` или евро `€`, затем ввести годовую аренду домена в той же валюте и выбрать округление. Годовая аренда домена делится на 12 и прибавляется к месячной аренде сервера перед делением между платными клиентами. По умолчанию сумма на клиента считается без округления до шага. При необходимости можно включить округление вверх до выбранного значения, например до `10`, `50` или `100`.

То же через команду:

```bash
xray-telegram payment-amount "500 ₽"
xray-telegram payment-domain-rent 1200
xray-telegram payment-rounding none
xray-telegram payment-rounding step 10
```

Сумма в напоминаниях считается как `(месячная аренда сервера + годовая аренда домена / 12) / количество платных клиентов`. Бесплатные клиенты в расчёт не входят.
Если в выводе видно `Total monthly rent: не указана`, значит ни месячная аренда сервера, ни годовая аренда домена ещё не сохранены, и `Amount per paid client` не сможет рассчитаться. В этом случае нужно снова открыть `Telegram бот -> Настроить оплату и округление` или выполнить `xray-telegram payment-amount "500 ₽"`.

Настроить реквизиты для сообщения об оплате:

```text
Telegram бот -> Настроить реквизиты оплаты
```

По умолчанию реквизиты не указываются, и сообщение остаётся в базовом виде. Можно выбрать перевод по номеру карты, на банковский счёт или по номеру телефона. Для телефона меню предложит банк из списка: `Т-Банк (Тинькофф)`, `Сбербанк`, `ВТБ`, `Альфа-Банк`, `Газпромбанк`, либо ручной ввод названия. Телефон сохраняется в нормализованном виде, например `+70000000000`, чтобы Telegram показывал его как номер.

То же через команду:

```bash
xray-telegram payment-details
xray-telegram payment-details none
xray-telegram payment-details phone "+70000000000" "BANK_NAME"
xray-telegram payment-details card "CARD_NUMBER"
xray-telegram payment-details bank-account "BANK_ACCOUNT_NUMBER"
```

Клиентская подписка работает так: пользователь пишет боту `/start`, и до привязки ссылки бот показывает только кнопки `Подключить уведомления` и `Помощь`. После кнопки `Подключить уведомления` пользователь отправляет VLESS-ссылку, затем менеджер сопоставляет её с клиентом по UUID, порту и параметрам подключения. Для Reality проверяются `pbk`, `sni`, `sid`, `fp`, `type`, `flow`; для TLS/XHTTP проверяются домен/SNI, `security=tls`, `type=xhttp`, `path`, `mode` и `extra`, если у подключения настроен XHTTP advanced profile. Часть ссылки после `#` игнорируется. Если пользователь уже подписан, повторный `/start` или обычное сообщение показывают текущую подписку, а меню содержит `Статус подписки`, `Получить VLESS-ссылку`, `Страна подключения`, `Статистика трафика`, `Помощь` и последней кнопкой `Отписаться от бота`. Telegram command menu работает так же: default-меню до подписки содержит только `/start` и `/help`, а для подписанных чатов отдельно включаются `/status`, `/link`, `/traffic` и `/unsubscribe`. После отписки бот отвечает `Подписка на бота отключена.` Напоминания отправляются в 08:00 по часовому поясу менеджера за 5 дней и за 1 день до `expiresAt`; для одного клиента, даты доступа и порога напоминание отправляется один раз.

Кнопка `Получить VLESS-ссылку` и fallback-команда `/link` отправляют пользователю актуальную ссылку из текущей конфигурации сервера. В Telegram ссылка приходит блоком кода, чтобы её было удобно скопировать одним нажатием. Это удобно после изменения порта, SNI, fingerprint, XHTTP path/mode, XHTTP advanced profile или других параметров подключения.

Кнопка `Страна подключения` показывает текущую выбранную страну и список доступных cascade-стран. При выборе другой страны бот запускает `xray-client route CLIENT TAG`, сохраняет выбор в SQLite и переключает runtime balancer через Xray API без изменения VLESS-ссылки. После смены страны пользователю нужно переподключить VPN, чтобы новые соединения пошли через выбранный маршрут. Если пользователь нажимает уже выбранную страну, бот отвечает, что эта страна уже выбрана.

Кнопка `Статистика трафика` и fallback-команда `/traffic` открывают подменю отчётов для подписанного клиента: за текущие сутки, за текущие сутки по часам и за последние 7 календарных дней по дням. Отчёты считаются в часовом поясе менеджера и не показывают пользователю внутреннее имя клиента.

Кнопка `Помощь` и fallback-команда `/help` показывают краткое описание возможностей бота:

```text
<ИМЯ_БОТА>: помощь

Я рядом, чтобы было проще следить за доступом к VPN.

Здесь можно:
• проверить статус подписки;
• получить актуальную VLESS-ссылку;
• сменить страну подключения;
• посмотреть статистику трафика;
• отписаться от бота, если уведомления больше не нужны.

После смены страны переподключи VPN, чтобы новые соединения пошли через выбранный маршрут.

Обычно я напоминаю об оплате в 08:00 за 5 дней и за 1 день до окончания доступа.

Иногда я также присылаю технические уведомления: например, если на сервере проводятся работы или возможны краткие перебои связи.
```

Если `/start` отправляет владелец бота, в клиентском меню дополнительно появляется кнопка `Админ-панель`. Админские кнопки работают только для сохранённого `chatId` владельца:

- `Статус` -> `Статус бота`, `Сводка сервера`, `Проверка сервера`, `Проверить напоминания`
- `Клиенты` -> `Добавить клиента`, `Получить VLESS-ссылку`, `Подписки клиентов`, `Продлить подписку`
- `Платежи` -> `Текущая сумма`, `Сумма на клиента`, `Округление`
- `Уведомления` -> `Плановые работы`, `Работы завершены`, `Новости`, `Своё сообщение`
- `Бэкапы` -> `Создать backup`
- `Активность` -> `Проверить GeoIP`
- `Настройки сервера` -> `TLS`
- `Настройки бота` -> `Статус бота`
- `Клиентское меню`

В разделе `/admin -> Клиенты` есть действие `Добавить клиента`. Бот попросит отправить имя и срок доступа одним сообщением, например `ivan 30`; если срок не указан или равен `0`, доступ будет бессрочным. Затем нужно выбрать статус оплаты. Если VLESS-подключений несколько, бот дополнительно покажет список подключений. После успешного добавления бот вернёт текст, который можно переслать пользователю: VPN-ключ в копируемом блоке кода, как при кнопке `Получить VLESS-ссылку`, подсказку подключить уведомления в боте, срок доступа, сумму оплаты для платного клиента и реквизиты перевода в том же формате, что и в напоминаниях об оплате. При привязке владельца `xray-telegram owner` сохраняет `botUsername` из Telegram `getMe`; если он сохранён, подсказка покажет кликабельный `@BOT_USERNAME`, иначе используется обычное имя бота.

Действие `/admin -> Клиенты -> Получить VLESS-ссылку` показывает список клиентов и после выбора запускает `xray-client link NAME`. Бот отправляет актуальную ссылку в HTML-блоке кода и заменяет фрагмент после `#` на имя сервера из `SERVER_NAME`, поэтому сообщение можно переслать клиенту, который потерял ключ и не подключал бота.

В разделе `/admin -> Клиенты` также есть действие `Продлить подписку`. После него бот покажет кнопки с клиентами, затем попросит отправить число дней. Введённое число передаётся в `xray-client extend-days NAME DAYS`, поэтому сохраняются обычные правила продления: если доступ ещё активен, дни добавляются к текущей будущей дате; если доступ истёк или был бессрочным, продление считается от текущей даты в часовом поясе менеджера.

Раздел `/admin -> Платежи` только показывает сохранённые настройки оплаты и расчёт суммы на платного клиента: месячную аренду сервера, годовую аренду домена, месячную долю домена и итоговую месячную сумму. Изменение суммы, валюты, годовой аренды домена, округления и реквизитов остаётся в SSH-меню `Telegram бот` и командах `xray-telegram payment-amount`, `xray-telegram payment-domain-rent`, `xray-telegram payment-rounding`, `xray-telegram payment-details`.

Раздел `/admin -> Настройки сервера -> TLS` показывает Caddy site configs, текущий TLS-профиль и время последнего изменения каждого site config. После выбора site config можно оперативно сменить профиль на `Caddy default`, `TLS 1.2`, `TLS 1.2 + TLS 1.3` или `TLS 1.3`. Бот меняет только TLS-директиву выбранного файла, поэтому это работает и для TLS/XHTTP reverse proxy, и для статического сайта без upstream local port. Перед изменением создаётся backup файла, затем выполняется `caddy validate` и reload Caddy; при ошибке предыдущий файл восстанавливается.

Админские и клиентские inline-кнопки защищены от повторных быстрых нажатий. После первого принятого нажатия старое сообщение с кнопками считается использованным; если Telegram позже принесёт ещё один callback с того же или более старого сообщения, бот ответит, что кнопка устарела, и действие не выполнит. Продолжать работу нужно с последнего сообщения бота.

Текст приветствия для клиента:

```text
Привет. Я бот <ИМЯ_БОТА>.

Я помогу не забыть о переводе за совместную аренду сервера.

Чтобы подключить напоминания, нажми кнопку «Подключить уведомления» или просто отправь сюда свою VLESS-ссылку. По ней я определю твой ключ и включу уведомления.
```

В автоматическом режиме ответы на сообщения пользователей обрабатывает `xray-telegram-poller.service`. Он держит исходящий long polling-запрос к Telegram API и отвечает почти сразу после сообщения пользователя. `xray-traffic-sync.timer` остаётся для фоновых задач: запускает `xray-telegram notify-geoip --quiet` и `xray-telegram notify-expiry --quiet`. Исключения suspicious из `manager.db` учитываются: по ним GeoIP-уведомления не отправляются. `notify-geoip` продолжает отправлять полный админский отчёт владельцу и дополнительно рассылает личные GeoIP-предупреждения только тем клиентским подпискам, где включены `Уведомления активности`. Личные исключения клиента хранятся в Telegram-состоянии и подавляют только клиентскую рассылку этого чата; админские отчёты и raw activity events остаются без изменений.

```text
<ИМЯ_БОТА>: напоминание об оплате

Через 5 дней заканчивается текущий период.
Доступ до: 2026-07-04 00:00 Europe/Moscow
Сумма оплаты: 500 ₽

```

Если реквизиты оплаты настроены по номеру телефона, сообщение дополнится так:

```text
+70000000000
Банк: BANK_NAME
```
